# adlix-consent — Konzept & Prozessdesign

> Grundlage: Soziokratie / Konsent-Prinzip
> Motto: *„Gut genug für jetzt — sicher genug zum Ausprobieren."*

---

## Begriffe

| Begriff | Bedeutung |
|---|---|
| **Vorhaben** | Ein Anliegen / eine Idee / ein Beschlussgegenstand (früher: Projekt/Aktion/Aktivität) |
| **Runde** | Ein Abstimmungsdurchlauf zu einem Vorhaben |
| **Konsent** | Abwesenheit schwerwiegender, begründeter Einwände |
| **Einwand** | Begründeter Einspruch, der das gemeinsame Ziel betrifft |
| **Integration** | Prozess zur Einarbeitung von Einwänden in den Vorschlag |

---

## Kern-Unterschied: Konsens vs. Konsent

- **Konsens**: Alle müssen aktiv zustimmen → oft blockiert
- **Konsent**: Niemand hat einen *schwerwiegenden, begründeten* Einwand → pragmatisch, schnell

---

## Haupt-Prozess (Consent-Loop)

```
1. VORHABEN EINREICHEN
   - Titel, Beschreibung, Ziel
   - Welche Spannung / welches Problem soll gelöst werden?
   - Kreis / Gruppe definieren (wer ist betroffen?)

2. INFORMATIONSRUNDE
   - Nur Verständnisfragen — keine Meinungen, keine Diskussion
   - Plattform filtert aktiv: Meinungen werden zurückgewiesen
   - Einreicher beantwortet Fragen; Antworten werden dokumentiert

3. REAKTIONSRUNDE (1. Meinungsrunde)
   - Jede Person äußert kurz ihre Perspektive — reihum
   - Kein Gegenargumentieren; aktives Zuhören
   - Zeitlimit pro Beitrag empfohlen

4. ANPASSUNG des Vorschlags
   - Einreicher kann Vorhaben auf Basis der Reaktionen überarbeiten
   - Änderungen werden versioniert und sichtbar gemacht

5. ABSTIMMUNGSRUNDE (Konsent-Runde)
   Optionen:
   ✅ Konsent           — Ich habe keinen Einwand
   💛 Leichter Einwand  — Anmerkung, trage die Entscheidung aber mit (kein Blocker)
   🔴 Schwerwiegender Einwand — Muss integriert werden (Blocker)
   ⏸️  Enthalten        — → Folgeprozess (siehe unten)

6a. KEIN schwerwiegender Einwand → Beschluss gefasst ✅
    - Evaluationsdatum setzen (wann wird neu geprüft?)
    - Leichte Einwände werden dokumentiert, können integriert werden

6b. SCHWERWIEGENDER EINWAND → Integration (siehe unten)
    - Einwand + Verbesserungsvorschlag einbringen
    - Diskussion & Integration
    - Vorschlag anpassen → neuer Loop ab Schritt 5

6c. ENTHALTEN → Folgeprozess Enthaltung (siehe unten)
```

---

## Folgeprozess: Enthaltung ⏸️

> Enthaltung ist möglich, aber nicht als bequemer Ausweg.
> Der Prozess stellt sicher, dass hinter einer Enthaltung keine versteckten Einwände stecken.

### Schritt 1: Grund abfragen (Pflichtfeld)

Die Person wählt einen Grund:

| Grund | Konsequenz |
|---|---|
| A — *Ich bin nicht betroffen / habe keine Expertise* | Enthaltung akzeptiert, kein weiterer Schritt |
| B — *Ich brauche mehr Informationen* | → Informationsanforderung (Schritt 2) |
| C — *Mir ist ein Teil des Vorhabens unklar* | → Klärungsanforderung (Schritt 2) |
| D — *Ich habe Bedenken, möchte sie aber nicht öffentlich äußern* | → Anonymisierungspfad (Schritt 3) |
| E — *Ich möchte mich nicht festlegen* | → Reflexionspfad (Schritt 4) |

---

### Schritt 2: Informations- / Klärungsanforderung (Grund B + C)

- Person formuliert konkret: *„Ich brauche Information zu: ..."*
- Anforderung geht an Einreicher / Kreis
- Einreicher hat X Tage Zeit zu antworten
- Nach Antwort: Person wird erneut zur Abstimmung eingeladen
- Bleibt die Antwort aus → Runde wird pausiert / Einreicher wird benachrichtigt

---

### Schritt 3: Anonymisierungspfad (Grund D)

> Ziel: Psychologische Sicherheit schaffen, damit echte Einwände auftauchen können.

- Person gibt Bedenken anonym ein
- **Pro-Feature (KI)**: KI fasst alle anonymen Bedenken thematisch zusammen
  - *„3 Personen haben Bedenken zum Thema Ressourcen, 2 zum Thema Zeitplan"*
  - Keine Rückschlüsse auf Einzelpersonen
- Zusammenfassung geht an Einreicher + Kreis
- Einreicher entscheidet: Vorhaben anpassen? Neue Informationsrunde?
- Person wird erneut eingeladen abzustimmen — weiterhin anonym möglich

**Wichtig:** Anonyme Bedenken sind kein Blocker, aber ein Signal. Der Einreicher ist eingeladen (nicht gezwungen), darauf einzugehen.

---

### Schritt 4: Reflexionspfad (Grund E — „will mich nicht festlegen")

> Das problematischste Szenario: Verantwortung wird vermieden.

Der Prozess ermutigt zur Reflexion — ohne zu zwingen:

**Schritt 4a: Soziokratischer Hinweis**
Die Plattform zeigt einen freundlichen, erklärenden Hinweis:
> *„In der Soziokratie trägt jede beteiligte Person Mitverantwortung für Entscheidungen.
> Enthaltung ohne Grund bedeutet: Du hast keinen Einwand.
> Hast du vielleicht doch einen — auch einen kleinen?"*

**Schritt 4b: Geführte Selbstreflexion (3 Fragen)**
Optionale, nicht erzwingbare Fragen:
1. *Gibt es einen Teil des Vorhabens, der dir Bauchschmerzen bereitet?*
2. *Würde das Vorhaben irgendjemanden betreffen, den du kennst — und wie?*
3. *Was bräuchtest du, um wirklich Konsent geben zu können?*

**Schritt 4c: Angebote**
- „Ich möchte trotzdem enthalten bleiben" → Enthaltung wird gespeichert (kein Blocker)
- „Ich habe doch einen leichten Einwand" → weiter zu 💛
- „Ich habe einen schwerwiegenden Einwand" → weiter zu 🔴
- „Ich möchte anonym Bedenken äußern" → weiter zu Anonymisierungspfad

**Schritt 4d: Analyse für den Einreicher (Pro)**
Wenn mehrere Personen sich mit Grund E enthalten:
- KI analysiert Muster: *„5 Enthaltungen ohne Begründung — mögliche Ursachen: Komplexität des Vorhabens, mangelnde Information, Konfliktvermeidung"*
- Empfehlung: *„Erwäge eine weitere Informationsrunde oder ein direktes Gespräch mit Betroffenen"*

## Schwerwiegender Einwand — Integrationsprozess

```
1. Person benennt Einwand (begründet, bezogen auf gemeinsames Ziel)
2. Plattform prüft Validität (Leitfragen):
   - Betrifft der Einwand das gemeinsame Ziel / den Auftrag des Kreises?
   - Ist es ein sachliches Argument — kein persönlicher Geschmack?
3. Person macht einen Verbesserungsvorschlag (wie könnte der Vorschlag angepasst werden?)
4. KI-Unterstützung (Pro, ein/ausschaltbar):
   - Fasst alle Einwände + Verbesserungsvorschläge zusammen
   - Generiert einen integrierten Entwurf, der alle Einwände adressiert
   - Zeigt transparent: „Einwand X wurde adressiert durch..."
   - Immer als Entwurf markiert — Menschen entscheiden
5. Einreicher passt Vorhaben an
6. Neuer Abstimmungsloop (ab Schritt 5 des Hauptprozesses)
```

## Abstimmungs-Signale im Überblick

| Signal | Bedeutung | Blocker? |
|---|---|---|
| ✅ Konsent | Kein Einwand | — |
| 💛 Leichter Einwand | Anmerkung, trage es mit | Nein |
| 🔴 Schwerwiegender Einwand | Muss integriert werden | Ja |
| ⏸️ Enthalten (A) | Nicht betroffen / keine Expertise | Nein |
| ⏸️ Enthalten (B/C) | Brauche mehr Info / Klärung | Temporär |
| ⏸️ Enthalten (D) | Anonyme Bedenken | Nein (aber Signal) |
| ⏸️ Enthalten (E) | Will mich nicht festlegen | Nein (aber Reflexionsprozess) |

---

## KI-Features (Pro, ein/ausschaltbar)

| Feature | Beschreibung |
|---|---|
| **Einwands-Synthese** | Fasst alle Einwände + Verbesserungsvorschläge zusammen |
| **Integrations-Entwurf** | Generiert angepassten Vorschlag aus Vorhaben + Einwänden |
| **Enthaltungs-Analyse** | Erkennt Muster bei mehrfachen Enthaltungen |
| **Anonyme Bedenken** | Aggregiert anonyme Inputs ohne Rückschluss auf Personen |
| **Prozess-Coach** | Hinweise an Moderator wenn Prozess stockt |

## Abgrenzung / Nicht-Ziele (aktuell)

- Kein ausführlicher Projektmanagement-Teil
- „Vorhaben" bleibt schlank: Titel + Beschreibung + Ziel + Kreis
- Keine Gantt-Charts, keine Aufgabenverwaltung
- Kein Projektsourcing (spätere Phase)
